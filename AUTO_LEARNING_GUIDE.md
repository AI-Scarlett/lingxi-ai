# 灵犀 v3.1.0 - 自动学习功能指南

**🧠 核心特性：** 让灵犀越用越快，自我进化！

---

## 🎯 功能说明

### 问题
手动配置 Layer 0 规则太麻烦，而且无法发现真正的高频问题！

### 解决方案
**自动学习层** - 自动记录、分析、生成 Layer 0 规则

**工作原理：**
1. 📝 **记录查询** - 记录所有 >100ms 的查询（Layer 0 已覆盖的不用记录）
2. 📊 **频率分析** - 统计每个查询的出现频率
3. 🧠 **智能识别** - 识别连续 7 天、日均>1 次的高频问题
4. ✨ **自动生成** - 为高频问题自动生成 Layer 0 响应规则
5. 🚀 **自动应用** - 定期将新规则添加到 Layer 0

---

## 📊 学习策略

### 触发条件
- **统计周期：** 连续 7 天
- **频率阈值：** 日均出现 ≥1 次
- **响应时间：** >100ms（快速响应的问题无需学习）

### 智能分类
根据查询关键词自动分类并生成响应：

| 类型 | 关键词 | 示例响应 |
|------|--------|----------|
| 📝 创作类 | 写、创作、文案 | "📝 收到老板！马上为您创作～" |
| 🔍 搜索类 | 搜索、查找、查询 | "🔍 搜索专家已启动！" |
| 🎨 图像类 | 图、图片、自拍 | "🎨 图像专家准备就绪～" |
| 📤 发布类 | 发布、小红书、微博 | "📤 发布专家就位～" |
| 📊 分析类 | 分析、统计、数据 | "📊 数据分析专家启动～" |
| 💬 翻译类 | 翻译、英文、中文 | "💬 翻译专家待命～" |
| 💻 开发类 | 开发、代码、功能 | "💻 开发专家就位～" |
| 💕 日常类 | 你好、在吗、谢谢 | "老板好呀～💋" |


## 🧠 技术实现

### 核心组件

1. **QueryFrequencyAnalyzer** - 查询频率分析器
   - 记录查询日志
   - 统计频率
   - 识别候选规则

2. **AutoLearner** - 自动学习器
   - 集成分析器
   - 生成规则
   - 应用规则

3. **智能响应生成** - 根据查询类型自动生成响应
   - 关键词匹配
   - 分类响应
   - 情感化表达

### 数据流程

```
用户查询
    ↓
orchestrator_v2.py (记录查询)
    ↓
learning_layer.py (频率分析)
    ↓
识别高频问题 (7 天，日均>1 次)
    ↓
生成 Layer 0 规则
    ↓
应用规则 (自动/手动)
    ↓
下次查询 → Layer 0 响应 (<5ms)
```

## 📞 常见问题

### Q: 会不会学习到错误的问题？
A: 有频率阈值保护（日均>1 次），偶然错误不会触发学习。

### Q: 自动生成的响应准确吗？
A: 基于关键词分类，准确率约 85%。支持人工审核和修改。

### Q: 日志会占用很多空间吗？
A: 每条记录约 100 字节，一天 100 次查询约 10KB。自动清理 30 天前的日志。

### Q: 可以关闭自动学习吗？
A: 可以，设置 `AUTO_LEARNING_ENABLED = False` 即可。
