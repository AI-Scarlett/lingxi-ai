## 📝 版本命名规则

### 主版本号 (MAJOR)
- 重大架构变革
- 不兼容的 API 修改
- 示例：v1.0.0 → v2.0.0

### 次版本号 (MINOR)
- 新增功能 (向下兼容)
- 重大优化
- 示例：v2.8.0 → v2.9.0

### 修订号 (PATCH)
- Bug 修复
- 小优化
- 示例：v2.8.3 → v2.8.4
