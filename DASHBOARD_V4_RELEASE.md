## 🎯 核心改进

### 1. 架构重构

**旧架构 (v3.3.3)**
```
server.py ──► 读取 HEARTBEAT.md (文本解析)
     │
     └──► 返回给 index.html
```

**新架构 (v4.0)**
```
main.py ──► SQLite 数据库 (完整数据层)
    │
    ├──► WebSocket 实时推送
    ├──► 内存缓存加速
    ├──► 自动同步 OpenClaw
    └──► index.html (Vue3 + ECharts)
```

## 🚀 快速启动

### 方式一：使用启动脚本

```bash
cd dashboard/v4
python start.py
```

### 方式二：使用 uvicorn

```bash
cd dashboard/v4/backend
uvicorn main:app --host 0.0.0.0 --port 8765 --reload
```

### 访问

```
http://localhost:8765?token=YOUR_TOKEN
```

## 🔧 技术栈

### 后端
- **FastAPI** - 高性能异步框架

## 📈 数据迁移

启动 v4 时会自动从 OpenClaw 同步历史数据：

```python
# 自动执行
1. 读取 ~/.openclaw/agents/main/sessions/sessions.json
2. 解析每个 session 的 .jsonl 文件
3. 提取用户消息创建任务记录
4. 写入 SQLite 数据库
```

手动触发同步：
```bash
curl -X POST "http://localhost:8765/api/sync?token=YOUR_TOKEN"
```

## 🐛 已知问题

1. **首次启动较慢** - 同步大量历史数据时需要等待
   - 解决：后续启动会跳过已同步的数据

2. **WebSocket 重连** - 网络不稳定时会自动重连
   - 解决：已实现自动重连机制
